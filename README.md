HOW TO ORGANIZE PROJECT AND VERSIONS

### NOMENCLATURE EXAMPLE (1.5.1.1)
- 1º = project version
- 2º = project sub version
- 3º = branch number. 1 for santo andre, 2 for lagoa santa
- 4º = release version
- 5º = in hotfix case, add date in this format (AAAAMMDD)
